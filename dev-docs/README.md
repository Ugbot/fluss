# dev-docs

Local-only scratch space. This directory is gitignored (see `.gitignore`) and
must never be committed or pushed upstream. Use it for:

- `adr/` — Architecture Decision Records (MADR-style, numbered sequentially).
- `design/` — longer-form design documents in progress.
- `research/` — raw notes, protocol traces, benchmark outputs, API matrices.

Files in here are not Apache Fluss documentation. Once a design is
ready to propose upstream, the authoritative version belongs in a GitHub
issue, PR description, or the Apache Fluss website repository — not here.
